## Criterion 2: resumability test asserts zero duplicate rows after a simulated restart

`test_resumability_no_duplicates_after_simulated_restart` (in the run above,
PASSED) writes only the first half of the fixture's rows (simulating a
process killed mid-stream), then "restarts" by writing the full fixture's
rows again (AISStream has no resume/backfill, so a restart reprocesses
everything available, per ADR 0001 Decision 3). It reads back every row from
the resulting Parquet partitions and asserts:

```python
keys = [(row["mmsi"], row["message_type"], row["payload_hash"]) for row in all_rows]
assert len(keys) == len(set(keys))
assert len(all_rows) == len(rows)
```

Both assertions passed: zero duplicate `(MMSI, message_type, payload_hash)`
tuples, and the final row count equals the fully-deduped input count.

## Known caveat, carried forward honestly rather than hidden

`tests/fixtures/aisstream_sample.jsonl` is a **synthetic** fixture generated
to match the field-name schema documented in `docs/data_sources.md`
(AISStream's public documentation prose plus the field-name contract pinned
in `SHIP_STATIC_DATA_REQUIRED_FIELDS`), not a message set captured from a
real AISStream subscription. No live network call was made or authorized
this session. This means:

- All 16 tests are valid unit/integration tests of the ingestion code's
  logic (parsing, dedup, partitioning, resumability, schema-drift guard,
  reconnect handling).
- The field names themselves (`ImoNumber`, `MaximumStaticDraught`, etc.)
  remain UNVERIFIED against real AISStream traffic, as recorded in
  `docs/data_sources.md`. If real field names differ, the schema-drift test
  will correctly fail loudly against a real fixture rather than silently
  passing bad data, which is the safety property M1 required.
- Recommended follow-up (not part of M1's acceptance criteria, tracked as an
  open item): capture one real subscription sample with the `.env` API key
  and replace the synthetic fixture, or add it alongside as a second
  verification fixture.
