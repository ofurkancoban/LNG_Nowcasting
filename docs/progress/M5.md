## Known simplifications, flagged rather than hidden

- `estimate_delivered_volume_cbm` requires the caller to supply
  `laden_draught_m` and `ballast_draught_m` explicitly. `data/reference/lng_carriers.csv`
  (M2) does not yet carry these columns, and M3's `PositionSample` does not
  carry draught at all, so this model function is self-contained rather than
  wired end-to-end into the M1-M3 pipeline. Wiring it up requires either
  extending the M2 reference schema or capturing draught in AIS position
  samples — neither was in M5's deliverable list, and doing so silently
  would have expanded already-committed milestones' scope.
- `APPROXIMATE_GWH_PER_CBM = 0.0060` is a rough, uncalibrated LNG
  energy-density constant used only to make cargo volumes comparable to
  ALSI's GWh/d send-out figures for backtest purposes. It is explicitly
  documented as needing real calibration before this model is trusted for
  anything beyond exercising the backtest harness's mechanics.
- The metrics Parquet schema denormalizes aggregate `mae`/`mape` as repeated
  columns across every per-fold row, rather than a separate summary table.
  This satisfies the "row count >= 20" and "has MAE/MAPE columns" criteria
  simply; a production version might prefer a separate one-row summary
  table alongside the per-fold detail table.
