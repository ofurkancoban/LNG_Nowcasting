## Criterion 5: `systemd-analyze verify` exits 0

This machine is macOS (no systemd). Verified inside a Debian container with
`systemd-analyze` installed (not requiring systemd as PID 1), with the unit's
referenced paths (`ExecStart` binary, `EnvironmentFile`, `User=`) stubbed out
so the check validates syntax and the `Restart=` policy rather than failing
on an environment-only path-existence mismatch:

```
$ docker run -d --name lng-systemd-verify debian:bookworm-slim sleep infinity
$ docker exec lng-systemd-verify bash -c "apt-get update -qq && apt-get install -y -qq systemd"
$ docker exec lng-systemd-verify bash -c "
    useradd -m lng
    mkdir -p /opt/lng-nowcasting/.venv/bin
    printf '#!/bin/sh\nexit 0\n' > /opt/lng-nowcasting/.venv/bin/python
    chmod +x /opt/lng-nowcasting/.venv/bin/python
    touch /opt/lng-nowcasting/.env
    chown -R lng:lng /opt/lng-nowcasting
  "
$ docker cp deploy/systemd/lng-ingest-aisstream.service \
    lng-systemd-verify:/etc/systemd/system/lng-ingest-aisstream.service
$ docker exec lng-systemd-verify systemd-analyze verify /etc/systemd/system/lng-ingest-aisstream.service
$ echo "REAL_EXIT=$?"
REAL_EXIT=0
$ docker rm -f lng-systemd-verify
```

The first attempt (without stubbing the target paths) genuinely failed with
`Command /opt/lng-nowcasting/.venv/bin/python is not executable: No such
file or directory` — a real signal from the tool, not a false pass, that
`systemd-analyze verify` checks referenced paths exist. Documented in
`deploy/README.md` so this is reproducible and the reasoning isn't lost.

## Criterion 6: VPS health-check workflow exists and fails on staleness

`.github/workflows/vps-ingester-health-check.yml` runs every 30 minutes,
SSHes into the VPS (`appleboy/ssh-action`, credentials via
`secrets.VPS_HOST`/`VPS_SSH_USER`/`VPS_SSH_PRIVATE_KEY` — never hardcoded),
finds the newest `*.parquet` file under
`/opt/lng-nowcasting/raw/source=aisstream`, and exits 1 if its age exceeds
`MAX_STALENESS_MINUTES=20` or if no file exists at all. This could not be
executed end to end against a real VPS in this session (no VPS access
attempted, matching the earlier security guidance not to use inline
credentials) but passes `actionlint` and its logic was reasoned through
explicitly above rather than left unverified without comment.
