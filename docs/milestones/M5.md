# M5: Nowcast model and backtest harness

## Objective

Estimate delivered cargo volumes from detected arrival events and validate
the resulting nowcast against vintaged GIE ALSI ground truth using
walk-forward backtesting.

## Deliverables

- `src/lng/nowcast/model.py` (cargo volume estimate from vessel capacity
  and draught change, aggregated to a terminal-level nowcast)
- `src/lng/nowcast/backtest.py` (walk-forward validation harness, reading
  vintaged ALSI snapshots as of each backtest date, per ADR 0001)
- `tests/test_nowcast_model.py`
- `tests/test_backtest.py`
- `marts/backtest/metrics_<run_id>.parquet` (output metrics artifact schema)

## Acceptance criteria

- `pytest tests/test_nowcast_model.py` passes offline against fixture-based
  arrival events.
- `pytest tests/test_backtest.py` passes and asserts the backtest harness
  only ever reads ALSI data with a `built_at` timestamp at or before the
  simulated "as of" date for each fold (no lookahead into future ALSI
  revisions), verifiable by asserting on the vintages selected.
- Backtest run produces a metrics file (`marts/backtest/metrics_*.parquet`)
  and a test asserts the row count meets a minimum sample-size threshold
  (e.g. `>= 20` terminal-days) before accuracy metrics are considered valid.
- Metrics file includes at minimum MAE and MAPE columns, checked by a
  pandera schema test.

## Out of scope

- Dashboard rendering of these metrics (M6).
- Live/production scheduling of backtest runs (M7).

## Dependencies

M2 (vessel capacity), M3 (arrival events), M4 (vintaged ALSI ground truth).
