# M6: Evidence.dev dashboard

## Objective

Publish a dashboard showing current nowcast estimates per terminal and a
backtest accuracy page sourced from the M5 metrics artifacts.

## Deliverables

- `dashboard/` Evidence.dev project (pages, sources config pointing at
  `marts/`)
- `dashboard/pages/index.md` (current nowcast overview)
- `dashboard/pages/backtest-accuracy.md` (backtest accuracy page reading
  `marts/backtest/metrics_*.parquet`)
- `dashboard/sources/` DuckDB source connection to the marts layer

## Acceptance criteria

- `npm run build` (or the Evidence-equivalent build command) inside
  `dashboard/` exits 0.
- The build output includes a generated route/file corresponding to
  `backtest-accuracy` (verifiable by checking the build output directory
  for the expected page path).
- The dashboard build fails loudly (non-zero exit) if
  `marts/backtest/metrics_*.parquet` does not exist or fails its pandera
  schema check, rather than silently rendering an empty page.

## Out of scope

- Authentication/access control for the dashboard.
- Historical vintage browsing UI (dashboard shows latest by default; vintage
  inspection remains a query-layer capability from ADR 0001, not necessarily
  surfaced in the UI at this milestone).

## Dependencies

M5 (backtest metrics artifacts must exist and have a stable schema).
