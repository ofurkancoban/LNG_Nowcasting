# M0: Repo scaffold and CI

## Objective

Establish a working Python project skeleton with dependency management,
linting/typing/testing wired together, and CI that runs them on every push.

## Deliverables

- `pyproject.toml` (project metadata, dependencies, tool config for ruff/mypy/pytest)
- `uv.lock`
- `Makefile` with a `check` target running ruff, mypy --strict, and pytest
- `.gitignore` excluding `raw/`, `staging/`, `marts/`, `.env`, `__pycache__/`, `.venv/`
- `.github/workflows/ci.yml` triggering on push and pull_request, running `make check`
- `src/lng/__init__.py`, `src/lng/config.py` (env-var based config loading)
- `tests/test_config.py` (at least one passing placeholder test)

## Acceptance criteria

- `uv sync` exits 0.
- `make check` exits 0 on a clean checkout.
- `ruff check src tests` exits 0.
- `mypy --strict src` exits 0.
- `pytest tests/` exits 0 with at least 1 test collected.
- `.github/workflows/ci.yml` contains an `on:` block with both `push` and
  `pull_request` triggers (verifiable with
  `grep -A2 '^on:' .github/workflows/ci.yml`).
- `git check-ignore raw staging marts .env` exits 0 for all four paths.

## Out of scope

- Any ingestion, vessel, event, or nowcast logic.
- Dashboard scaffolding.
- Secrets/credentials management beyond reading from environment variables.

## Dependencies

None. This is the first milestone.
