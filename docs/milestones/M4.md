# M4: GIE ALSI ingestion as ground truth

## Objective

Ingest daily GIE ALSI send-out and inventory data per terminal as vintaged,
immutable ground truth for backtesting.

## Deliverables

- `src/lng/ingest/alsi.py` (ALSI API client: `x-key` auth, pagination
  handling, `type=eu`/`country`/`facility` params per docs/data_sources.md)
- `tests/fixtures/alsi_sample_response.json`
- `tests/test_ingest_alsi.py`
- Vintage snapshot writer implementing ADR 0001-architecture.md Decision 4
  (`built_at`-stamped snapshots, no in-place overwrite)

## Acceptance criteria

- `pytest tests/test_ingest_alsi.py` passes fully offline against the
  recorded fixture.
- A test asserts the ingestion client never issues more than one polling
  request per calendar day per facility under normal operation (enforcing
  the "data only updates once/day at 19:30 CET" fact from
  docs/data_sources.md in code, not just as a comment).
- A test asserts that re-ingesting the same gas day writes a new vintage
  snapshot rather than overwriting the previous one, and that both vintages
  remain independently queryable afterward.
- A pandera schema test validates required ALSI fields
  (`gasDayStart`, `inventory`, `sendOut`, `dtmi`, `dtrs`, status flag) are
  present and correctly typed in ingested output.

## Out of scope

- Nowcast modeling or backtesting logic (M5).
- Any AIS-derived data.

## Dependencies

M0 (repo scaffold). Independent of M1-M3 (parallel-buildable).
