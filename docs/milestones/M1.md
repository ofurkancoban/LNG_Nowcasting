# M1: AIS ingestion to raw parquet

## Objective

Ingest AISStream.io WebSocket messages into append-only, timestamp-partitioned
raw Parquet, resumable across restarts, with all tests running against a
recorded fixture rather than a live connection.

## Deliverables

- `src/lng/ingest/aisstream.py` (WebSocket client, subscription setup,
  message writer)
- `tests/fixtures/aisstream_sample.jsonl` (real captured messages, including
  at least one `PositionReport` and one `ShipStaticData` message, with the
  actual observed field names — this fixture is also the schema
  verification artifact called out as UNVERIFIED in docs/data_sources.md)
- `tests/test_ingest_aisstream.py`
- `docs/data_sources.md` updated to mark the AISStream Type 5 field names as
  VERIFIED once the fixture is captured, or to record the corrected field
  names if they differ from the documentation's prose description

## Acceptance criteria

- `pytest tests/test_ingest_aisstream.py` passes fully offline (no network
  access), using only `tests/fixtures/aisstream_sample.jsonl`.
- A resumability test simulates a process restart mid-stream (write raw
  files, kill, restart ingestion against the same fixture) and asserts zero
  duplicate rows exist for any `(MMSI, message_type, payload_hash)` tuple in
  the resulting raw partition.
- A schema-drift test asserts the fixture's `ShipStaticData` message
  contains all fields the ingestion code reads by name, failing loudly if a
  field is missing (guards against the AISStream schema-drift risk in
  docs/risks.md).
- Raw output partitioning matches ADR 0001-architecture.md Decision 1
  (`source=aisstream/ingest_date=.../hour=...`), verifiable by inspecting
  the output directory structure after a fixture-driven test run.

## Out of scope

- Vessel identification or LNG-carrier classification (M2).
- Geofencing or event detection (M3).
- Any live network call in the test suite.

## Dependencies

M0 (repo scaffold, `make check`).
