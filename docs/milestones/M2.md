# M2: Vessel registry and LNG carrier identification

## Objective

Build a vessel registry that identifies which MMSI/IMO numbers are LNG
carriers and estimates their cargo capacity, resolving the open data-source
question from docs/data_sources.md.

## Deliverables

- `src/lng/vessels/registry.py` (MMSI/IMO lookup, LNG carrier flag)
- `src/lng/vessels/capacity.py` (cargo capacity in cbm per identified vessel)
- `data/reference/lng_carriers.csv` (curated static reference table:
  IMO, name, cargo capacity cbm, build year, propulsion type), or
  documented integration with whatever external source the user approves
  per the open question in docs/decisions/0001-architecture.md
- `tests/fixtures/vessel_registry_labeled_sample.json` (small labeled set of
  known LNG carriers and known non-LNG tankers by MMSI/IMO)
- `tests/test_vessel_registry.py`

## Acceptance criteria

- `pytest tests/test_vessel_registry.py` passes offline against the labeled
  fixture.
- Classification precision and recall against the labeled fixture are both
  computed and asserted above an explicit numeric threshold in the test
  (e.g. `assert precision >= 0.95` and `assert recall >= 0.90`), not just
  "looks right."
- Every vessel in `data/reference/lng_carriers.csv` has a non-null cargo
  capacity value, checked by a pandera schema test.
- Reference table source and licence are documented in a header comment or
  accompanying `data/reference/README.md`, consistent with the licence
  findings in docs/data_sources.md (no CC BY-NC-only source used without an
  explicit non-commercial-use acknowledgment).

## Out of scope

- Geofencing or event detection (M3).
- Real-time enrichment of the AIS stream with registry data (deferred to M3
  integration).

## Dependencies

M1 (raw AIS ingestion, for the ShipStaticData IMO field).
